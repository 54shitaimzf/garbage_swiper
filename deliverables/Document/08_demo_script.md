# 5-10 分钟答辩演示脚本

> 原则：先展示稳定、可重复的功能，再介绍扩展架构；所有未完成项按测试报告如实说明。

## 0:00-1:00｜场景与架构

说明园区巡检问题、Jetson 边缘计算、手机 PWA、8081 手动链路和 8082 自主入口。点出 ROS2、SLAM、融合和 MQTT 的扩展位置。

## 1:00-3:00｜底盘控制与安全

打开 8081，依次演示前进、后退、左移、右移、旋转、调速和急停；松开摇杆后展示停车。此段对应手机控制、通信和安全要求。

## 3:00-5:00｜边缘 AI 与告警

展示低帧率视频和 `best.engine` 识别结果；放置演示目标，展示类别/置信度事件，再用手动鸣笛验证告警。优先低速，出现异常立即急停。

## 5:00-7:00｜自主任务入口

打开 8082，展示地图列表、任务状态、停止和返回入口。若 ROS2/Nav2 尚未完成真车验收，应明确说明这里是 mock API 演示，不启动真实运动。

## 7:00-10:00｜工程能力与收尾

展示目录结构、自动化测试、CI 检查、模式隔离和 MQTT 可选接口；最后回到 8081，确认急停和手动链路可恢复。
