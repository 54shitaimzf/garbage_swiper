# 项目叙事与定位

## 一句话介绍

本项目面向校园/园区道路巡检场景，使用 Jetson 边缘计算小车完成移动控制、摄像头采集、`best.engine` 目标识别和本地告警，并为 ROS2、SLAM/Nav2、多传感器融合和 MQTT 云端扩展预留标准接口。

## 核心闭环

```text
手机/PWA 控制
    ↓
8081 手动网关 → 官方 TCP 6000 → Rosmaster 底盘
    ↓
摄像头 → best.engine → 识别结果/置信度告警/蜂鸣器
```

8082 是独立的自主任务入口，当前用于验证地图选择、任务状态和安全停止流程；真实 ROS2/Nav2 接管必须单独完成传感器、地图、TF 和运动安全验收。

## 项目价值

- 将重复的园区巡检工作转化为可操作、可观察的移动平台任务。
- 使用边缘推理减少对持续联网的依赖。
- 使用 ROS2 标准话题和独立服务，为雷达、里程计、相机、导航和云端事件扩展留出空间。
- 采用手机浏览器/PWA，降低演示部署门槛；当前不依赖 Android Studio 或鸿蒙设备。
