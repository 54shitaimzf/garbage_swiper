# 项目需求分析

## 1. 建设目标

针对校园/园区道路巡检重复度高、覆盖范围分散和异常反馈不及时的问题，构建一套“移动平台 + 边缘视觉 + 标准机器人接口”的智能巡检终端。项目以稳定可演示为优先，并保留向自主导航和云端协同扩展的能力。

## 2. 基础要求对照

| 基础要求 | 当前实现与证据 | 状态 |
| --- | --- | --- |
| ROS2 核心通信与分布式通信 | `ros2_ws/`、`icar_perception`、`icar_autonomy`，预留 `/scan`、`/odom`、`/tf`、图像和检测接口 | 🟡 接口/源码就绪，真车运行待验收 |
| SLAM 建图与自主导航 | 8082 地图/任务 API、地图白名单和路线配置；已有地图优先的集成方案 | 🟡 方案就绪，Nav2/真车路线待验收 |
| AI 边缘推理与多传感器融合 | 车端 `best.engine`、YOLO 告警链路；雷达/深度/视觉决策级融合接口已规划 | 🟡 AI链路可演示，真实融合待验收 |
| 云平台与 CI/CD | `cloud/` 可选 MQTT 接口、`.github/` 自动检查、单元测试和协议回归 | 🟡 本地验证完成，Broker 联调待补 |
| 手机 APP 控制 | 8081 手机 PWA，支持摇杆、方向、调速、视频、告警和急停 | ✅ 已实现并作为主演示链路 |

## 3. 功能需求

- 控制：前进、后退、左移、右移、旋转、调速、松手停车和急停。
- 感知：低帧率视频回传、`best.engine` 检测、置信度事件和本地告警。
- 机器人接口：读取雷达、里程计、TF、RGB/深度数据，并为导航和融合保留标准话题。
- 任务：地图选择、任务启动、停止和返回入口；真实运动必须经过资源仲裁和安全检查。
- 工程：代码、测试、部署、回滚和文档可追溯，模型和密钥不进入仓库。

## 4. 非功能要求

安全优先、手动链路零回归、模式资源独占、默认离线可用、失败安全停车、所有未验证能力如实标注。
