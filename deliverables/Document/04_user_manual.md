# 项目使用手册

## 1. 开机与联网

1. 给小车供电并确认电量充足。
2. 手机和 Jetson 连接同一个手机热点。
3. 在车端执行 `hostname -I` 获取当前热点 IP。
4. 首次测试让车轮悬空，速度从低值开始，急停保持可操作。

## 2. 8081 手动控制

在手机浏览器打开：

```text
http://<小车IP>:8081/
```

地址栏只填写 IP，不填写 `http://`；端口填写 `8081`。依次验证前进、后退、左移、右移、旋转、调速、松手和急停。

## 3. 视频、识别与告警

8081 使用车端已有 `best.engine` 推理链路。先确认视频正常，再放置演示目标，观察类别和置信度事件；手动鸣笛先在车轮悬空状态验证，随后再低速实车测试。

## 4. 8082 自主任务入口

打开：

```text
http://<小车IP>:8082/
```

当前入口默认用于 mock 状态验证，不直接驱动电机。未完成 ROS2/Nav2 资源检查前，不应将 mock 任务描述为真实自主导航。

## 5. 常见故障

- 连接失败：确认同一热点、重新获取 IP、端口填写正确。
- 页面可开但不动作：先急停，检查 6000 和 8081；不要同时启动 ROS2 底盘驱动。
- 摄像头异常：停止重复启动的相机/YOLO进程，恢复原 8081 启动脚本。
- 负载过高：立即停车，保留日志，不要反复启动 TensorRT 进程。
